get

get retrieves any property from any Batman.Object. Instead of doing normal property lookups like you would in normal Javascript, you must use get to retrieve anything from a Batman object.

get a simple key

js> this.obj.get('foo')
bar

get a key on a smart object

js> this.obj.get('baz')
true

Note that while the above works, doing a vanilla JavaScript key access will not! 

js> this.obj['baz']
undefined
js> this.obj.baz
undefined

This is because @obj has some special keys on it, which can only be accessed through the Batman runtime. Using the runtime is as simple as always using get and set to retrieve or set properties on Batman.Objects. This is a departure from the usual way of doing things, and it takes more code, but it's the only way that the rest of the goodness Batman provides can be implemented.

set

set stores a value in a property on any Batman.Object. Instead of doing normal sets like you would in normal Javascript, you must use set to retrieve anything from a Batman object.

set a simple key

js> this.obj.set('foo', 'bar')
bar
js> this.obj.get('foo')
bar

set a key on a smart object

js> this.obj.set('baz', 'qux')
qux

Note that while the above works, doing a vanilla JavaScript key access will not! 

js> this.obj['baz']
undefined
js> this.obj.baz
undefined
js> this.obj.get('baz')
undefined

This is because @obj has some special keys on it, which can only be accessed through the Batman runtime. Using the runtime is as simple as always using get and set to retrieve or set properties on Batman.Objects. This is a departure from the usual way of doing things, and it takes more code, but it's the only way that the rest of the goodness Batman provides can be implemented.

observe
